# 技术栈说明（C++ + RPC + Redis + MySQL + Docker）

## 1. 技术选型总览

| 领域 | 技术 | 用途 |
|---|---|---|
| 开发语言 | C++17 | 服务端核心开发 |
| 构建工具 | CMake | 跨平台构建管理 |
| RPC | gRPC + Protocol Buffers | 服务间通信与接口定义 |
| 缓存 | Redis 7 | 热点数据缓存与快速查询 |
| 数据库 | MySQL 8 | 结构化数据持久化 |
| 容器化 | Docker + Docker Compose | 本地一键部署与环境统一 |
| 日志 | spdlog（建议） | 结构化日志输出 |
| MySQL 驱动 | mysql-connector-c++（建议） | C++ 访问 MySQL |
| Redis 客户端 | redis-plus-plus（建议） | C++ 访问 Redis |

## 2. 各技术在项目中的职责

### 2.1 gRPC + Protobuf
- 用 `.proto` 描述服务接口与消息结构
- 自动生成 C++ stub，减少手写序列化逻辑
- 支持清晰的服务边界，便于后续拆分扩展

### 2.2 Redis
- 缓存高频查询（用户信息、任务列表）
- 降低数据库读取压力
- 用 TTL 或主动失效策略保证数据一致性

### 2.3 MySQL
- 存储用户与任务主数据
- 通过索引保证查询性能
- 事务保证关键写入的一致性

### 2.4 Docker / Docker Compose
- 封装运行环境，避免“本地可运行、线上不可运行”
- 一条命令拉起多服务，适合学习和联调
- 可扩展到 CI 或测试环境

## 3. 推荐版本
- `C++`: 17 或以上
- `CMake`: 3.20+
- `gRPC`: 1.5x+
- `Protobuf`: 与 gRPC 匹配版本
- `Redis`: 7.x
- `MySQL`: 8.0+
- `Docker`: 24+
- `Docker Compose`: v2+

## 4. 工程分层建议
- `api/proto`：RPC 协议定义
- `handler`：RPC 请求入口，参数校验
- `service`：业务逻辑
- `repository`：数据库访问
- `cache`：缓存访问封装
- `infra`：配置、日志、连接池等基础设施

## 5. 数据流说明（示例）
1. 客户端调用 `task_service.CreateTask`
2. `task_service` 通过 RPC 调用 `user_service.GetUser` 校验用户
3. 校验通过后写入 MySQL
4. 删除或刷新 Redis 的任务列表缓存
5. 返回创建结果

## 6. 本地开发依赖（建议）
```bash
# build
sudo apt-get install -y build-essential cmake pkg-config

# docker
sudo apt-get install -y docker.io docker-compose-plugin

# project
# gRPC / Protobuf / mysql-connector-c++ / redis-plus-plus
```

## 7. Docker Compose 服务建议
```yaml
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: demo
    ports:
      - "3306:3306"

  redis:
    image: redis:7
    ports:
      - "6379:6379"

  user_service:
    build: ./services/user_service
    depends_on:
      - mysql
      - redis
    ports:
      - "50051:50051"

  task_service:
    build: ./services/task_service
    depends_on:
      - mysql
      - redis
      - user_service
    ports:
      - "50052:50052"
```

## 8. 风险与注意事项
- gRPC 与 Protobuf 版本要严格匹配
- 缓存一致性优先用“先写库、再删缓存”策略
- MySQL 初始化脚本要纳入版本管理
- Docker 镜像构建要固定依赖版本，避免环境漂移

## 9. 后续可扩展方向
- 引入服务注册与配置中心
- 增加链路追踪（OpenTelemetry）
- 增加鉴权（JWT / API Gateway）
- 引入 CI 自动化测试与镜像构建
